# Abstract Workflow

Use this note when the task is abstract-specific rather than full-manuscript writing.

## Recommended order

1. Decide whether the main problem is structure, claim strength, significance framing, or language.
2. If the diagnosis is unclear, run an abstract audit before rewriting.
3. If the core finding is buried, change the order before changing the style.
4. If the ending overclaims, soften it before polishing.
5. If the core logic already works, then run a polish pass.

## Common output modes

- diagnostic audit first
- one strong rewrite plus one safer rewrite
- claim-softening options
- polish-only pass

## Boundary rules

- Do not invent data, statistics, or mechanism support.
- Do not silently expand an abstract task into full-manuscript service.
- If the user asked only for an abstract, keep the scope there.
